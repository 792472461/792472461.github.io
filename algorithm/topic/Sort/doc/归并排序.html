归并排序 #

归并排序整体思路 #

// 伪代码
function mergeSort(arr, l, r) {
  if(l >= r) return
  const mid = (l + r) / 2
  // 对mid[l, mid]排序
  mergeSort(arr, l, mid)
  // 对mid(mid + 1, r)排序
  mergeSort(arr, mid + 1, r)
  // 合并数组,主要代码就是在这里
  merge(arr, l, mid, r)
}

总结 #

  1. 时间复杂度：归并排序法最好情况是O(n)级别的时间复杂度，完全随机的是O(n log n)
  2. 空间复杂度：因为归并排序法需要额外空间，所以空间复杂度是O(n)
  3. 稳定性：稳定
  4. 优化：1. 判断是否需要merge 2. 对小规模数据使用插入排序 3. 只创建一个临时空间
